# Результаты выполнения тестового задания

## Задача

Тестовое задание выполнено для позиции «Вайбкодер-аутричер» в Polza Agency.

Цель задания — показать практический подход к сбору B2B-базы, персонализации, подготовке цепочки cold email-писем и автоматизации части рутины через скрипт.

## Что сделано

1. Собрана база из 50 B2B-компаний.
2. Для каждой компании добавлены:
   - название;
   - сайт;
   - ниша;
   - город;
   - контактное лицо или роль;
   - email;
   - источник email;
   - краткий факт о компании;
   - персонализация;
   - источник факта;
   - комментарий.
3. Подготовлена цепочка из 3 cold email-писем:
   - письмо 1 — холодное знакомство с переменной `{{персонализация}}`;
   - письмо 2 — follow-up через 3 дня с другим углом;
   - письмо 3 — финальный follow-up через 5 дней с мягким выходом.
4. Подготовлен Python-скрипт `polza_personalize_companies.py`.
5. Подготовлен файл `polza_vibe_stack.txt` с описанием инструментов и подхода.
6. Добавлены файлы результата работы скрипта.

## Где находится результат

Основной результат оформлен в Google Таблице с двумя листами:

- `База` — список компаний, контакты, источники, факты и персонализация.
- `Письма` — цепочка из 3 писем.

В GitHub-репозитории находятся:

- `polza_personalize_companies.py` — скрипт персонализации и проверки;
- `polza_companies.csv` — CSV-версия исходной базы;
- `companies_with_personalization.csv` — результат работы скрипта в стандартном CSV-формате;
- `validation_report.csv` — отчёт проверки в стандартном CSV-формате;
- `companies_with_personalization_excel.csv` — версия результата с разделителем `;` для удобного открытия в русской версии Excel;
- `validation_report_excel.csv` — версия отчёта с разделителем `;` для удобного открытия в русской версии Excel;
- `polza_vibe_stack.txt` — описание vibe-coding стека;
- `README.md` — краткое описание проекта;
- `RESULTS.md` — этот отчёт.

## Как работает скрипт

Скрипт читает файл:

```text
polza_companies.csv
```

На выходе создаёт:

```text
companies_with_personalization.csv
validation_report.csv
companies_with_personalization_excel.csv
validation_report_excel.csv
```

Файлы с `_excel` используют разделитель `;`, чтобы их можно было открыть в русской версии Excel двойным кликом и сразу увидеть данные по колонкам.

Скрипт выполняет следующие действия:

1. Проверяет наличие обязательных колонок.
2. Читает данные компаний из CSV.
3. Проверяет формат email.
4. Проверяет, заполнена ли колонка `Персонализация`.
5. Если персонализация уже заполнена вручную — не перетирает её.
6. Если персонализация пустая — генерирует короткий персонализированный текст на основе компании, ниши и краткого факта.
7. Создаёт итоговый CSV-файл с результатом.
8. Создаёт отдельный отчёт `validation_report.csv`.
9. Создаёт дополнительные Excel-friendly версии файлов с разделителем `;`.

## Команда запуска

```bash
python polza_personalize_companies.py
```

Дополнительно можно запустить проверку сайтов:

```bash
python polza_personalize_companies.py --check-sites
```

## Комментарий по базе

База собрана по B2B-компаниям из России и смежных рынков: CRM, SaaS, digital, маркетинг, коммуникации, аналитика, сервисы для бизнеса.

Email взяты из публичных источников, указанных в таблице.

SMTP/deliverability-проверка почтовых ящиков отдельно не выполнялась, поэтому email указаны как публично найденные контакты, а не как дополнительно валидированные через отдельный сервис.

## Комментарий по персонализации

Персонализация сделана не как абстрактный шаблон, а через краткий факт о компании или продукте.

Формат персонализации:

- что делает компания;
- почему это связано с B2B-аутричем;
- какой потенциальный угол можно использовать для outbound-гипотезы.

## Комментарий по письмам

Цепочка писем сделана в спокойном B2B-тоне:

- без агрессивной продажи;
- без обещаний «увеличить продажи в 3 раза»;
- с фокусом на проверку гипотез, сегментов и аккуратный outbound;
- с мягким завершением в третьем письме.

## Почему такой подход

Я сделал решение простым и воспроизводимым:

- база хранится в Google Таблице;
- CSV можно отдельно проверить и обработать скриптом;
- скрипт не требует платных API;
- ручная персонализация не теряется;
- результат можно проверить через отдельный отчёт;
- Excel-friendly версии позволяют быстро посмотреть результат глазами;
- подход можно быстро расширить под реальные процессы агентства.
