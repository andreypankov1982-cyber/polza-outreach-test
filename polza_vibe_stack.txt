Вайбкод-стек для тестового задания Polza Agency

IDE / окружение:
- Windows PowerShell
- Python 3.x, стандартная библиотека без внешних зависимостей
- Google Таблицы
- GitHub
- ChatGPT / Codex-подход для декомпозиции, генерации структуры, проверки и ускорения рутины

Что делал через AI:
- декомпозиция тестового задания на шаги;
- структура базы и колонок;
- подготовка логики персонализации;
- Python-скрипт для генерации персонализации, проверки обязательных полей, email-формата и создания отчёта;
- цепочка из 3 писем для cold email outreach;
- оформление README и RESULTS.md для понятной сдачи результата.

Как работает скрипт:
- на вход принимает файл polza_companies.csv;
- читает колонки: Компания, Сайт, Ниша, Email, Краткий факт, Персонализация, Источник факта;
- проверяет наличие обязательных колонок;
- проверяет формат email;
- если персонализация пустая — генерирует короткий персонализированный текст;
- если персонализация уже заполнена вручную — не перетирает её;
- создаёт стандартные CSV-файлы:
  - companies_with_personalization.csv
  - validation_report.csv
- создаёт Excel-friendly версии этих файлов с разделителем ";":
  - companies_with_personalization_excel.csv
  - validation_report_excel.csv

Почему такой подход:
- он простой, проверяемый и воспроизводимый;
- не требует платных API и внешних зависимостей;
- показывает механику автоматизации рутинной персонализации;
- сохраняет ручную персонализацию и не перетирает уже заполненные данные;
- создаёт отдельный отчёт проверки, чтобы быстро увидеть проблемы по строкам;
- дополнительно создаёт Excel-friendly версии для удобной ручной проверки результата;
- при необходимости его можно расширить: добавить парсинг сайтов, LLM API, проверку deliverability email, enriched data и интеграцию с Google Sheets.
